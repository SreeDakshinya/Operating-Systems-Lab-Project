PAGE REPLACEMENT ALGORITHMS

A Page Replacement Algorithm is a technique employed by OS to manage memory by selecting which pages to replace in the main memory when it becomes full. The Primary goal is to Minimize page faults and optimize system performance. Various algorithms like FIFO, Optimal, LRU, MRU and Random are used, each with its own strategy for determining the page to evict. The choice of algorithm depends on system requirements and workload characteristics.

FIFO

FIFO (First-In-First-Out) page replacement is a fundamental algorithm in operating systems' memory management. It works by replacing the oldest page in memory when a new page needs to be loaded, based on the order of their arrival. Pages are stored in a queue-like structure, with the page that entered memory earliest residing at the front of the queue. When memory is full and a new page must be loaded, FIFO evicts the page at the front of the queue, maintaining the order of arrival. However, FIFO can suffer from the "Belady's anomaly," where increasing the number of page frames can lead to an increase in page faults.

OPTIMAL

The Optimal page replacement algorithm is an idealized method used in memory management. It replaces the page that will not be used for the longest period of time in the future. This algorithm requires knowledge of future page accesses, which is not practical in real-world systems but serves as a benchmark for comparison with other algorithms. Optimal page replacement minimizes the number of page faults by always selecting the best candidate for eviction based on future reference patterns. However, its practical implementation is limited due to the impossibility of predicting future memory accesses accurately.

LRU

LRU (Least Recently Used) is a page replacement algorithm commonly used in operating systems. It works on the principle that pages that have been least recently accessed are more likely to be least used in the near future. When a page needs to be replaced and memory is full, LRU evicts the page that has not been accessed for the longest time. This algorithm requires maintaining a record of when each page was last accessed, typically done using a data structure like a queue or a linked list. While effective in reducing the number of page faults by prioritizing recently used pages, LRU can be computationally expensive to implement in systems with a large number of pages.

MRU

The MRU (Most Recently Used) page replacement algorithm is a technique used in operating systems to manage memory by replacing the page that has been most recently accessed. It assumes that recently used pages are more likely to be accessed again in the near future.

RANDOM

The Random page replacement algorithm selects a page to be replaced randomly from the set of pages in memory when a new page needs to be loaded and memory is full. It does not consider the usage history or access patterns of pages, making its selection entirely based on chance. While simple to implement, Random does not prioritize pages based on their likelihood of future use, which can lead to unpredictable performance. This algorithm may result in a relatively high number of page faults compared to more sophisticated algorithms like LRU or Optimal, especially in scenarios with specific access patterns or memory usage characteristics.
